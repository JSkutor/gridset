# GridSet Keyboard UX Reference

GridSet은 루틴 편집, 실시간 기록, 로그 확인까지 키보드 중심으로 이동하고 입력하도록 설계되어 있다. 이 문서는 사용자가 키보드로 수행할 수 있는 모든 주요 작업, 포커스 규칙, 예외 조건, UX 의도를 한곳에 정리한다.

## 1. 전역 키보드 정책

### 1.1 단축키가 비활성화되는 상황

다음 모달이 열려 있으면 전역 단축키는 동작하지 않는다.

- 온보딩 모달
- 도움말 및 데이터 관리 모달
- 인증 모달

사용자가 모달 안에서 확인, 취소, 데이터 초기화 같은 작업을 할 때 뒤쪽 화면의 탭 전환이나 그리드 포커스가 실수로 실행되지 않게 막는다.

### 1.2 입력 중에는 전역 단축키를 무시

전역 탭 전환 단축키는 다음 요소에 포커스가 있을 때 동작하지 않는다.

- `input`
- `textarea`
- `select`
- `contenteditable`

예를 들어 운동 이름 검색창에서 `W`를 입력해도 실시간 기록 탭으로 이동하지 않는다. 이름 편집, 메모 작성, 숫자 입력 중인 사용자의 의도를 우선한다.

### 1.3 포커스 해제

| 키 | 동작 | UX 의도 |
| --- | --- | --- |
| `Esc` | 현재 포커스된 요소를 blur 처리 | 입력 필드나 버튼 포커스를 빠져나와 전역 단축키를 다시 쓸 수 있게 함 |

단, 일부 화면 안에서는 `Esc`가 더 가까운 맥락을 먼저 처리한다. 예를 들어 루틴 이름/세션 이름 편집 중에는 편집 취소로 동작하고, 운동 검색 추가 행에서는 검색 취소 및 추가 행 닫기로 동작한다.

### 1.4 키보드 네비게이션 상태

화살표 키 또는 `Tab`으로 이동하면 `keyboard-navigating` 상태가 활성화된다. 이 상태에서는 마우스 hover 스타일이 남아서 현재 키보드 포커스를 방해하지 않도록 hover 효과를 억제한다. 마우스를 움직이거나 클릭하면 다시 마우스 사용 상태로 돌아간다.

## 2. 최상위 화면 전환

상단 네비게이션은 `Routine`, `Set`, `Log` 세 영역으로 구성된다.

| 키 | 이동 대상 | 화면 ID | 설명 |
| --- | --- | --- | --- |
| `Q` | 루틴 편집 | `R` | 루틴, 세션, 운동 구성과 목표값을 편집 |
| `W` | 실시간 기록 | `S` | 운동 세트의 중량/횟수 기록 |
| `E` | 로그 | `L` | 일일 로그, 운동별 추이, 루틴 타임라인 확인 |
| `Cmd + ←` / `Ctrl + ←` | 이전 최상위 화면 | 순환 | `R ← S ← L` 방향으로 이동 |
| `Cmd + →` / `Ctrl + →` | 다음 최상위 화면 | 순환 | `R → S → L` 방향으로 이동 |

최상위 화면 전환은 뷰 트랜지션을 사용해 방향감 있는 화면 이동으로 표현된다. 상단 네비게이션 버튼에 포커스가 있던 상태에서 전환하면 새 탭 버튼으로 포커스를 유지하려고 시도한다.

### 2.1 더 이상 쓰지 않는 숫자 단축키

| 키 | 현재 동작 |
| --- | --- |
| `1`, `2`, `3` | 전역 네비게이션으로 사용하지 않으며, 입력이 전파되지 않도록 조용히 차단 |

이전 단축키 습관이 의도치 않은 동작을 만들지 않도록 막는 호환 처리다.

## 3. 포커스 진입 및 전환

| 키 | 현재 화면 | 동작 |
| --- | --- | --- |
| `` ` `` / `₩` | 실시간 기록 화면 | 그리드로 진입, 그리드 안에서는 메모와 셀 포커스 토글 |
| `` ` `` / `₩` | 루틴 편집 화면 | 첫 세션의 첫 운동으로 포커스 진입. 운동이 없으면 첫 세션 행으로 포커스 |

맥/한국어 키보드 환경에서 백틱 위치가 `₩`로 입력될 수 있으므로 같은 진입 키로 취급한다.

실시간 기록 화면에서 이 키는 세 가지 상태를 구분한다.

- 그리드 밖: 마지막으로 사용한 셀 또는 첫 셀로 포커스
- 그리드 셀 안: 하단 세트 메모 textarea로 포커스
- 메모 안: 마지막으로 사용한 그리드 셀로 포커스

## 4. 실시간 기록 화면 키보드 작업

실시간 기록 화면은 스프레드시트처럼 중량과 횟수를 빠르게 입력하는 데 초점을 둔다. 기본 컬럼은 `kg`, `Reps` 두 개다.

### 4.1 그리드 셀 이동

| 키 | 동작 | 세부 규칙 |
| --- | --- | --- |
| `↑` | 같은 컬럼의 이전 세트로 이동 | 첫 행에서는 더 위로 이동하지 않음 |
| `↓` | 같은 컬럼의 다음 세트로 이동 | 마지막 행에서는 더 아래로 이동하지 않음 |
| `Enter` | 같은 컬럼의 다음 세트로 이동 | 세로 입력 흐름을 빠르게 이어감 |
| `Tab` | 다음 셀로 이동 | `kg → Reps → 다음 행 kg` 순서 |
| `Shift + Tab` | 이전 셀로 이동 | `Reps → kg → 이전 행 Reps` 순서 |
| `←` | 커서가 셀 맨 앞이면 이전 셀로 이동 | 텍스트 중간에서는 기본 커서 이동 유지 |
| `→` | 커서가 셀 맨 뒤이면 다음 셀로 이동 | 텍스트 중간에서는 기본 커서 이동 유지 |

좌우 화살표는 숫자 수정 중 커서 이동을 방해하지 않기 위해 입력값의 시작/끝에서만 셀 이동으로 바뀐다.

### 4.2 세트 행 추가

| 조작 | 조건 | 동작 |
| --- | --- | --- |
| `세트 추가` 버튼 | 계획보다 세트를 더 수행해야 함 | 해당 운동 블록에 빈 세트 행 추가 |

현재 사용자-facing 동작 기준으로 마지막 셀에서 `Tab`을 눌러 새 세트 행을 자동 추가하지 않는다. `Tab`은 그리드 안에서 다음 셀/다음 포커스 대상으로 이동하는 역할이다.

### 4.3 첫 중량 자동 복사

| 키 | 조건 | 동작 |
| --- | --- | --- |
| `Tab` | 각 운동의 첫 세트 `kg` 칸에서 중량 입력 후 이동 | 해당 운동의 나머지 세트 중량에 같은 값을 채우는 흐름으로 연결 |

반복 세트에서 같은 중량을 입력하는 수고를 줄인다. 이후 각 행의 중량은 필요하면 개별 수정할 수 있다.

### 4.4 세트 완료 및 휴식 타이머

| 키 | 조건 | 동작 |
| --- | --- | --- |
| `Tab` | `Reps` 칸에서 횟수 입력 후 이동 | 세트 완료 처리 흐름과 휴식 타이머 시작 흐름으로 연결 |

세트 간 휴식 시간과 운동 후 휴식 시간은 루틴 편집 화면의 운동 설정값을 따른다. 기록 입력을 마치는 순간 다음 행동이 자동으로 이어지도록 설계되어 있다.

### 4.5 세트 메모

| 키 | 동작 |
| --- | --- |
| `` ` `` / `₩` | 현재 포커스된 세트의 하단 메모 textarea와 그리드 셀 사이를 토글 |

메모는 전체 운동 메모가 아니라 현재 포커스된 세트에 귀속된다. 사용자가 어느 세트를 기록 중이었는지 `focusedSet` 상태로 추적한다.

### 4.6 운동 완료 저장

| 키 | 조건 | 동작 |
| --- | --- | --- |
| `Enter` | `운동 완료` 버튼에 포커스 | 운동 기록 저장 |

저장 가능한 데이터와 시작 시간이 있을 때 버튼이 활성화된다. 저장 성공 후에는 선택 세션과 활성 운동 포커스를 초기화하고 로그 화면으로 이동한다.

### 4.7 세션 선택 드롭다운

실시간 기록 화면 상단의 세션 선택은 기본 `select` 요소다. 브라우저/OS 기본 키보드 동작을 따른다.

- `Tab`: 드롭다운으로 포커스 이동
- `↑` / `↓`: 옵션 이동
- `Enter` 또는 브라우저 기본 확정 키: 선택

## 5. 루틴 편집 화면 키보드 작업

루틴 편집 화면은 세 개의 세로 패널로 나뉜다.

- 왼쪽: 세션 목록
- 가운데: 선택 세션의 운동 목록
- 오른쪽: 선택 운동의 설정

주요 UX 모델은 “왼쪽에서 오른쪽으로 들어가고, 왼쪽 화살표/Esc로 한 단계 빠져나오기”다.

### 5.1 루틴 편집 화면 진입

| 키 | 동작 |
| --- | --- |
| `` ` `` / `₩` | 첫 세션을 선택하고 첫 운동 행에 포커스. 운동이 없으면 첫 세션 행에 포커스 |

### 5.2 세션 목록

세션 행은 `tabIndex=0`으로 포커스 가능하다.

| 키 | 동작 |
| --- | --- |
| `↑` | 이전 세션으로 이동하고 선택 |
| `↓` | 다음 세션으로 이동하고 선택 |
| `↓` | 마지막 세션에서 누르면 `세션 추가` 버튼으로 이동 |
| `→` | 선택 세션의 첫 운동으로 이동. 운동이 없으면 `운동 추가` 버튼으로 이동 |
| `Enter` | 현재 세션 선택 |
| `Space` | 현재 세션 선택 |
| `Cmd + ↑` / `Ctrl + ↑` | 현재 세션을 위로 재정렬 |
| `Cmd + ↓` / `Ctrl + ↓` | 현재 세션을 아래로 재정렬 |

재정렬 후에는 같은 세션 ID로 다시 포커스를 잡는다. 순서가 바뀌어도 사용자가 “방금 움직인 항목”을 잃지 않게 하기 위한 처리다.

### 5.3 세션 추가 버튼

| 키 | 동작 |
| --- | --- |
| `↑` | 마지막 세션 행으로 돌아감 |
| `→` | 선택 세션의 첫 운동으로 이동. 운동이 없으면 `운동 추가` 버튼으로 이동 |
| `Enter` / `Space` | 브라우저 기본 버튼 동작으로 세션 추가 |

새 세션을 만들면 이름 편집 input에 자동 포커스가 들어간다.

### 5.4 세션 이름 편집

| 키 | 동작 |
| --- | --- |
| `Enter` | 세션 이름 저장 |
| `Esc` | 편집 취소 |

새로 만든 세션의 이름 편집을 취소하면 생성한 세션을 제거하고 이전 위치 또는 세션 추가 버튼으로 포커스를 복귀한다.

### 5.5 운동 목록

운동 행도 `tabIndex=0`으로 포커스 가능하다.

| 키 | 동작 |
| --- | --- |
| `↑` | 이전 운동으로 이동 |
| `↓` | 다음 운동으로 이동 |
| `↓` | 마지막 운동에서 누르면 `운동 추가` 버튼으로 이동 |
| `←` | 현재 세션 행으로 돌아감 |
| `→` | 오른쪽 설정 패널의 첫 설정값으로 이동 |
| `Enter` | 현재 운동 선택/선택 해제 토글 |
| `Space` | 현재 운동 선택/선택 해제 토글 |
| `Cmd + ↑` / `Ctrl + ↑` | 현재 운동을 위로 재정렬 |
| `Cmd + ↓` / `Ctrl + ↓` | 현재 운동을 아래로 재정렬 |

운동 재정렬 후에도 같은 운동 ID에 포커스를 복귀한다.

### 5.6 운동 추가 버튼

| 키 | 동작 |
| --- | --- |
| `↑` | 마지막 운동 행으로 돌아감 |
| `←` | 현재 세션 행으로 돌아감 |
| `Enter` / `Space` | 브라우저 기본 버튼 동작으로 운동 검색 추가 행 열기 |

운동 추가 행이 열리면 검색 input에 자동 포커스가 들어간다.

### 5.7 운동 검색 자동완성

| 키 | 조건 | 동작 |
| --- | --- | --- |
| `↑` | 검색 결과가 열려 있음 | 이전 검색 결과로 이동. 맨 위에서는 맨 아래로 순환 |
| `↓` | 검색 결과가 열려 있음 | 다음 검색 결과로 이동. 맨 아래에서는 맨 위로 순환 |
| `Enter` | 검색 결과가 있고 항목이 선택됨 | 선택된 운동을 세션에 추가 |
| `Enter` | 검색 결과가 없음 | 입력한 이름으로 커스텀 운동 추가 |
| `Esc` | 언제든 | 검색 팝업 닫기, input blur, 운동 추가 취소 |

IME 조합 중에는 `Enter` 처리를 무시해 한글 입력 확정이 중복 등록으로 이어지지 않게 한다.

### 5.8 운동 설정 패널

설정 패널의 값 표시 영역은 포커스 가능한 컨트롤이다. 목표 세트, 목표 횟수, 세트 간 휴식, 운동 후 휴식은 `spinbutton` 역할을 가지며, 좌우 구분은 커스텀 운동일 때만 포커스 가능한 버튼 역할이다.

| 키 | 동작 |
| --- | --- |
| `↑` | 이전 설정 컨트롤로 이동 |
| `↓` | 다음 설정 컨트롤로 이동 |
| `Cmd + ↑` / `Ctrl + ↑` | 현재 설정값 증가 |
| `Cmd + ↓` / `Ctrl + ↓` | 현재 설정값 감소 |
| `←` | 선택된 운동 행으로 돌아감 |
| `Esc` | 선택된 운동 행으로 돌아감 |

설정값 변화 단위는 컨트롤마다 다르다.

| 설정 | 증가/감소 단위 | 범위 |
| --- | --- | --- |
| 목표 세트 | 1세트 | 1-20 |
| 목표 횟수 | 1회 | 1-999 |
| 세트 간 휴식 | 15초 | 0-600초 |
| 운동 후 휴식 | 15초 | 0-600초 |
| 좌우 구분 | Both/L/R 토글 | 커스텀 운동만 가능 |

### 5.9 루틴 이름 편집

루틴 헤더의 편집 버튼으로 루틴 이름 편집 input에 들어간 뒤에는 다음 키를 쓴다.

| 키 | 동작 |
| --- | --- |
| `Enter` | 루틴 이름 저장 |
| `Esc` | 편집 취소 |

### 5.10 일반 Tab 순환

루틴 편집 화면에서 `Tab` / `Shift + Tab`은 브라우저 기본 포커스 순서를 따른다. 패널 간 빠른 의미 이동은 주로 `←` / `→`와 `` ` `` / `₩` 진입 키로 제공된다.

## 6. 로그 화면 키보드 작업

로그 화면에는 오른쪽 사이드바 서브탭이 있다. 로그 화면이 활성 상태일 때만 서브탭 단축키가 동작한다.

| 키 | 보기 | 설명 |
| --- | --- | --- |
| `A` | 일일 | 달력과 하루 기록 |
| `S` | 운동별 | 종목별 추이와 전체 기록 |
| `D` | 루틴 로그 | 루틴 시작과 구성 변화 |

로그 서브탭 전환은 최상위 화면 전환과 달리 별도의 뷰 트랜지션 없이 즉시 바뀐다. 사이드바 버튼에는 `aria-keyshortcuts`가 지정되어 보조기술이 단축키 정보를 알 수 있다.

## 7. 휴식 타이머 키보드 접근

휴식 타이머 자체에는 별도 전역 단축키가 없다. 타이머가 표시되면 버튼을 `Tab`으로 탐색하고 브라우저 기본 버튼 키로 조작한다.

| 대상 | 키 | 동작 |
| --- | --- | --- |
| 일시정지/재개 버튼 | `Enter` / `Space` | 휴식 타이머 일시정지 또는 재개 |
| 닫기 버튼 | `Enter` / `Space` | 타이머 닫기 |

타이머는 `aria-live="polite"`로 남은 시간을 갱신한다.

## 8. 도움말, 온보딩, 데이터 관리 모달

모달이 열려 있는 동안에는 뒤쪽 화면의 전역 단축키가 모두 멈춘다.

| 키 | 동작 |
| --- | --- |
| `Tab` / `Shift + Tab` | 모달 내부의 버튼과 입력 가능한 요소 사이 이동 |
| `Enter` / `Space` | 포커스된 버튼 실행 |

현재 구현상 모달 닫기는 화면의 닫기 버튼 클릭 또는 포커스 후 버튼 실행을 통해 수행한다. `Esc`로 모달을 닫는 전용 처리는 없다.

## 9. 접근성 및 사용성 메모

- 네비게이션 버튼과 로그 사이드바 버튼에는 `aria-keyshortcuts`가 지정되어 있다.
- 운동 설정의 숫자형 값은 `role="spinbutton"`과 `aria-valuemin`, `aria-valuemax`, `aria-valuenow`를 제공한다.
- 좌우 구분 설정은 커스텀 운동이 아닐 때 `tabIndex=-1`로 키보드 포커스 대상에서 제외된다.
- 키보드 이동 시 포커스된 행이나 설정 컨트롤은 `scrollIntoView`로 화면 안에 유지된다.
- 재정렬 작업 후에는 인덱스가 아니라 ID 기준으로 포커스를 복구한다.
- 입력 필드에서는 문자 입력을 우선하므로 전역 단축키 충돌을 최소화한다.

## 10. 사용자 작업별 빠른 흐름

### 10.1 키보드만으로 운동 기록 시작

1. `W`로 실시간 기록 화면 이동
2. 세션 드롭다운이 필요하면 `Tab`으로 이동해 세션 선택
3. `` ` `` / `₩`로 그리드 진입
4. `kg` 입력 후 `Tab`
5. `Reps` 입력 후 `Tab`
6. `Enter`, `↑`, `↓`, `Tab`으로 다음 세트 계속 기록
7. 필요 시 `` ` `` / `₩`로 세트 메모 작성
8. 마지막에 `운동 완료` 버튼으로 이동해 `Enter`

### 10.2 키보드만으로 루틴 구성 수정

1. `Q`로 루틴 편집 화면 이동
2. `` ` `` / `₩`로 세션/운동 목록 진입
3. 세션 목록에서 `↑` / `↓`로 세션 선택
4. `→`로 운동 목록 이동
5. 운동 목록에서 `↑` / `↓`로 운동 선택
6. `→`로 설정 패널 이동
7. `Cmd/Ctrl + ↑/↓`로 목표값 변경
8. `←` 또는 `Esc`로 운동 목록 복귀
9. 필요 시 `Cmd/Ctrl + ↑/↓`로 세션 또는 운동 순서 재정렬

### 10.3 키보드만으로 로그 확인

1. `E`로 로그 화면 이동
2. `A`로 일일 로그 확인
3. `S`로 운동별 추이 확인
4. `D`로 루틴 로그 확인
5. `Q` 또는 `W`로 루틴 편집/기록 화면 복귀
